3 상신 전 30초 체크리스트

  • 회계일자 = 증빙 작성일자 실제 결제일이 아니라 세금계산서/영수증에 찍힌 날짜 기준. 7월 결제·6월 증빙이면 6월로 작성.
  • 협조 라인 담당자 확인 법인카드·개인경비 → 우지훈 프로(차후 강지우 과장) / 보통예금 집행 → 임옥선 팀장. 혼용 시 반려.
  • 원인보고서 연동 확인 참조문서란에 직접 링크를 걸거나, 첨부파일의 🔗 링크 아이콘으로 공유 링크를 생성해도 연동으로 인정됨. 단, 파일만 올리고 링크 생성 없이 두거나 댓글로 제목만 언급하는 것은 인정되지 않음.
  • 수신참조에 경영지원팀 포함 권한이 없으면 원인보고서 검색·열람 자체가 불가능함.
  • 프로젝트 코드 확인 팀장 워크샵 등 공통 경비는 '아이지 전사 공통' 코드 사용.
  • 지결 금액·코드·일자 ↔ 전표 데이터 재대조 지결과 전표 불일치는 회계팀이 가장 자주 지적하는 항목. 상신 전 본인이 한 번 더 대조.

4 증빙 처리 시 자주 걸리는 문제 Q&A (확장판)

앞서 다룬 4가지(날짜·협조라인·참조문서·열람권한) 외에, 증빙 자체에서 반복적으로 발생하는 유형입니다. 해외 출장·구매가 많은 해외팀 특성을 반영했습니다.

  • Q5. 세금계산서 금액과 실제 카드 청구액이 1~2백 원 차이나면? 공급가액·부가세를 증빙 그대로 분리 입력하고, 원 단위 절사 차이는 지출내역 비고에 사유를 남기세요. 임의로 반올림해 맞추면 전표 대사에서 걸립니다.
  • Q6. 해외 출장비를 법인카드/계좌이체로 결제했는데 환율은 어떤 기준인가요? 법인카드는 카드사가 실제로 청구한 원화 금액을 그대로 사용, 계좌이체·현금은 지급일 매매기준율을 적용합니다. 임의 추정 환율 사용 시 반려됩니다.
  • Q7. 3만 원 초과 건인데 간이영수증만 받았어요. 세금계산서·계산서·신용카드매출전표·현금영수증 중 하나가 원칙적으로 필요합니다. 불가피한 경우 사유서를 첨부하고 상신 전에 회계팀과 먼저 협의하세요.
  • Q8. 출장 사전기안서 금액과 실제 지출액이 다른 채로 올려도 되나요? 차액이 있으면 참조문서로 건 원인보고서에 변경 사유를 함께 명시하세요. 소명 없이 초과분만 상신하면 반려됩니다.
  • Q9. 숙박비·식비·교통비를 한 건으로 합쳐서 올려도 되나요? 용도별합계 항목을 각각 나눠 입력해야 전표와 매칭됩니다. 뭉쳐서 올리면 회계팀에서 항목을 쪼개 재작업을 요청하게 됩니다.
  • Q10. 담당 → 협조 → 결재 → 시행 순서를 건너뛰고 결재만 먼저 받으면? 시스템이 순서를 강제하지 않는 경우가 있어 실수가 잦습니다. 상신 전 결재선 미리보기에서 순서를 한 번 확인하는 습관이 필요합니다.
  • Q11. 댓글에 원인보고서 제목을 적어뒀는데, 그것도 연동으로 인정되나요? 댓글 자체는 인정되지 않습니다. 다만 이번 건처럼 첨부파일에 🔗 링크 아이콘으로 공유 링크가 생성되어 있다면, 그 링크가 참조문서 연동과 동일하게 인정됩니다. 즉 "댓글로 제목만 언급"과 "첨부파일에 링크를 생성"은 다르게 취급됩니다 — 후자만 유효합니다.

5 반복 오류를 줄이는 시스템화 제안

"다시 하세요"라는 반려가 반복되는 이유는 개인의 부주의보다 구조가 오류를 막아주지 못하기 때문인 경우가 많습니다. 아래는 입력 부서와 회계팀 양쪽의 반복 작업을 줄이기 위한 구조적 제안입니다.

제안 1 증빙 유형 필수값 강제

3만 원 초과 건에 세금계산서류가 첨부되지 않으면 상신 버튼 자체가 비활성화되도록 결의서 양식에 조건을 건다. 사람이 매번 기억하지 않아도 시스템이 걸러준다.

제안 2 참조문서 표기 개선

실제로는 첨부파일 링크로 연동되어 있어도 참조문서란에는 "선택된 문서가 없습니다"라고 표시되어 혼동을 준다. 연동이 완료되면 참조문서란 문구도 함께 갱신되도록 경영지원팀에 화면 개선을 요청하면, 작성자와 회계팀 모두 헷갈리지 않는다.

제안 3 반려 사유 코드화

회계팀이 반려할 때 "R-01 날짜불일치 / R-02 참조문서누락 / R-03 협조라인오류" 같은 표준 코드를 선택하게 한다. 사유가 명확해지고, 코드별 통계를 내면 어느 부서가 어떤 유형에서 반복되는지 데이터로 드러난다.

제안 4 최신본 가이드 고정 배치

회계팀에 "자주 반려되는 사례를 정리한 최종 최신본 가이드"를 공지해달라고 역제안하고, 그 문서를 결의서 상신 화면에 링크로 고정한다. 공지가 흩어지지 않고 상신하는 순간 바로 보인다.

제안 5 상신 전 셀프 체크 도구

이 페이지 같은 체크리스트를 팀 공용 북마크로 두고, 결의서 작성 담당자가 상신 전 30초만 대조하도록 습관화한다. 회계팀 반려 전에 1차로 걸러지는 효과가 있다.

제안 6 월 1회 반려 유형 리뷰

회계팀-현업 부서가 15분씩 반려 코드 통계를 같이 보는 짧은 리뷰를 정례화한다. 특정 부서·특정 유형이 반복되면 그 지점만 집중적으로 손본다.
